# Lucas 后续 TODO

## 记忆系统升级

### 现状

当前是一个基础的三层记忆系统（`agents/memory.py`）：
- 对话上下文：内存级，最近 5 轮，原文截断
- 用户偏好：`memory/preferences.yaml`，扁平快照覆盖写入
- 历史结论：`memory/conclusions.jsonl`，子串匹配检索

### 问题

1. **检索全靠子串匹配**：结论和 wiki 的 `_find_wiki_context` 都是字符级匹配，没有语义理解。"电池龙头"找不到"宁德时代"，"新能源车"匹配不到"比亚迪"
2. **对话上下文无压缩**：5 轮之前的信息直接丢失，长对话中早期的重要决策会被遗忘
3. **偏好没有时间维度**：每次覆盖写入，不知道用户兴趣变化趋势（比如从关注新能源转向半导体）
4. **结论没有权重和衰减**：3 个月前的过时分析和昨天的分析同等对待，可能误导 Manager

### 升级方向

#### P0：语义检索（影响最大）
- 引入 embedding 模型（可以用 Gemini embedding 或本地模型）
- 结论和 wiki 内容向量化存储
- 用户提问时算余弦相似度召回，替代子串匹配
- 存储方案：轻量级用 numpy + json，后续可换 chromadb

#### P1：对话摘要压缩
- 超过 5 轮时，用 LLM 把早期对话压缩成一段摘要
- 保留关键信息（讨论了哪些股票、得出了什么结论、用户表达了什么偏好）
- 类似 ChatGPT 的 "running summary" 机制

#### P2：偏好时间序列
- 偏好不再覆盖写入，改为追加记录 `{date, event, change}`
- 可以回溯用户兴趣变化
- Manager 可以主动提醒："你之前关注新能源比较多，最近转向半导体了？"

#### P3：结论时效性衰减
- 给结论加权重，按时间衰减（比如半衰期 30 天）
- 检索时综合 相似度 × 时效权重 排序
- 过期结论标记为 stale，Manager 可以建议"上次分析已经过了一个月，要不要重新看看"
